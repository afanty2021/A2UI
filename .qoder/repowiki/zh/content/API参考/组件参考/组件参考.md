## Button（按钮）

### 视觉功能描述
按钮组件用于触发用户交互操作，通常包含一个文本或图标子组件，点击后会触发预定义的客户端动作。

### 属性（Props）
| 属性名 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `component` | `"Button"` | 必填 | 组件类型标识 |
| `child` | `string` | 必填 | 子组件ID，通常指向一个Text或Icon组件 |
| `primary` | `boolean` | `false` | 是否为高亮主按钮样式 |
| `action` | `object` | 必填 | 点击时触发的动作，包含`name`和可选的`context`上下文 |

### 事件（Events）
- **`a2ui.action`**: 点击按钮时触发，载荷包含`action`对象、`dataContextPath`和`sourceComponentId`

### 插槽（Slots）
- `<slot></slot>`: 用于渲染子组件（如Text或Icon）

### 使用示例（Lit）
```json
{
  "component": "Button",
  "id": "submit-btn",
  "child": "submit-text",
  "primary": true,
  "action": {
    "name": "submitForm"
  }
}
```

### 使用示例（Angular）
```json
{
  "component": "Button",
  "id": "login-btn",
  "child": "login-text",
  "action": {
    "name": "login",
    "context": {
      "userId": "user123"
    }
  }
}
```

### 无障碍（a11y）支持
- 内置`<button>`元素，天然支持键盘导航和屏幕阅读器
- 子组件文本自动作为按钮标签

### 响应式行为
- 使用Flex布局，可随容器缩放
- 支持通过`weight`属性控制在布局中的相对大小

## Card（卡片）

### 视觉功能描述
卡片组件用于将内容组织在具有边框和背景的容器中，常用于突出显示重要信息。

### 属性（Props）
| 属性名 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `component` | `"Card"` | 必填 | 组件类型标识 |
| `child` | `string` | 必填 | 容器内子组件的ID，若需多个子组件，需用Row/Column包裹 |

### 事件（Events）
- 无直接触发事件

### 插槽（Slots）
- `<slot></slot>`: 用于渲染单个子组件

### 使用示例（Lit）
```json
{
  "component": "Card",
  "id": "profile-card",
  "child": "profile-content"
}
```

### 使用示例（Angular）
```json
{
  "component": "Card",
  "id": "info-card",
  "child": "info-text"
}
```

### 无障碍（a11y）支持
- 语义化容器，不干扰内部组件的可访问性
- 支持ARIA标签通过主题系统扩展

### 响应式行为
- 自适应高度，内容溢出时自动滚动
- 在Flex布局中可伸缩

## TextField（文本框）

### 视觉功能描述
文本框组件用于接收用户文本输入，支持多种输入类型和客户端验证。

### 属性（Props）
| 属性名 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `component` | `"TextField"` | 必填 | 组件类型标识 |
| `label` | `stringOrPath` | 必填 | 输入框标签文本 |
| `text` | `stringOrPath` | 可选 | 当前输入值，支持绑定数据模型路径 |
| `usageHint` | `"longText" \| "number" \| "shortText" \| "obscured"` | 可选 | 输入类型提示，影响UI表现 |
| `validationRegexp` | `string` | 可选 | 用于客户端验证的正则表达式 |

### 事件（Events）
- **输入事件**: 当用户输入时，自动更新绑定的数据模型

### 插槽（Slots）
- 无插槽，内部使用`<input>`元素

### 使用示例（Lit）
```json
{
  "component": "TextField",
  "id": "email-input",
  "label": "邮箱地址",
  "usageHint": "shortText",
  "validationRegexp": "^[^@]+@[^@]+\\.[^@]+$"
}
```

### 使用示例（Angular）
```json
{
  "component": "TextField",
  "id": "password-input",
  "label": "密码",
  "usageHint": "obscured"
}
```

### 无障碍（a11y）支持
- 标准`<input>`元素，完整支持键盘和屏幕阅读器
- `label`属性自动关联输入框

### 响应式行为
- 宽度100%填充父容器
- 高度自适应

| `value` | `stringOrPath` | 必填 | 选中的日期时间值，ISO 8601格式 |
| `enableDate` | `boolean` | `true` | 是否允许选择日期 |
| `enableTime` | `boolean` | `true` | 是否允许选择时间 |

| `label` | `stringOrPath` | 可选 | 输入框标签 |

### 事件（Events）
- **输入事件**: 当用户选择日期时间时，自动更新绑定的数据模型

### 插槽（Slots）
- 无插槽

### 使用示例（Lit）
```json
{
  "component": "DateTimeInput",
  "id": "meeting-time",
  "label": "会议时间",
  "enableDate": true,
  "enableTime": true,
  "value": ""
}
```

### 使用示例（Angular）
```json
{
  "component": "DateTimeInput",
  "id": "birth-date",
  "label": "出生日期",
  "enableTime": false,

}
```

### 无障碍（a11y）支持
- 支持键盘导航和日期选择

### 响应式行为

```json
{
  "component": "ChoicePicker",
  "id": "color-picker",
  "label": "选择颜色",
  "usageHint": "mutuallyExclusive",
  "options": [
    { "label": "红色", "value": "red" },
    { "label": "蓝色", "value": "blue" }
```

### 视觉功能描述

### 属性（Props）
| 属性名 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `component` | `"Tabs"` | 必填 | 组件类型标识 |
| `tabItems` | `array` | 必填 | 标签项数组，每项包含`title`和`child`（内容组件ID） |

### 事件（Events）
- 无直接触发事件，切换由用户点击标签触发

### 插槽（Slots）
- 通过`tabItems`中的`child`引用间接使用

### 使用示例（Lit）
```json
{
  "component": "Tabs",
  "id": "profile-tabs",
  "tabItems": [
    { "title": "基本信息", "child": "basic-info" },
    { "title": "设置", "child": "settings" }
  ]
}
```

### 使用示例（Angular）
```json
{
  "component": "Tabs",
  "id": "app-tabs",
  "tabItems": [
    { "title": "主页", "child": "home-panel" },
    { "title": "消息", "child": "messages-panel" }
  ]
}
```

### 无障碍（a11y）支持
- 应实现ARIA标签（`role="tablist"`等）
- 支持键盘导航（Tab键切换）
- 当前选中标签应有明确视觉指示

### 响应式行为
- 在小屏幕上可能转换为下拉选择
- 标签可滚动（overflow）
